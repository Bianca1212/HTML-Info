Git: O privire de ansamblu

Git este un sistem de control al versiunilor distribuit, utilizat pe scară largă pentru a gestiona proiectele de software. Spre deosebire de sistemele centralizate tradiționale, Git stochează un istoric complet al modificărilor fiecărui fișier din cadrul depozitului tău local, oferind o flexibilitate și o fiabilitate sporite.

Caracteristici cheie ale Git:

  • Urmărirea versiunilor: Urmărește fiecare modificare adusă fișierelor din proiectul tău, creând un istoric detaliat al dezvoltării acestuia.
  • Ramuri (Branch-uri): Permite crearea de fluxuri de lucru paralele, experimentând noi funcții sau remediind erori fără a afecta codul principal.
  • Contopire (Merge): Combină cu ușurință modificările din diferite ramuri, rezolvând conflictele dintre ele.
  • Colaborare: Permite mai multor dezvoltatori să lucreze simultan la același proiect, facilitând comunicarea și contribuția.
  • Controlul local: Oferă un depozit complet al proiectului pe computerul tău, permițând lucrul offline și accesul la istoric chiar și fără conexiune la internet.
  • Performanță: Este rapid și eficient, optimizat pentru proiecte de mari dimensiuni.
  • Open-source: Este un software liber și open-source, cu o comunitate vastă de utilizatori și contribuitori.

Beneficiile utilizării Git:

  • Menține controlul: Urmărește modificările, revine la versiuni anterioare și fii mereu la curent cu stadiul proiectului.
  • Crește calitatea codului: Urmărește evoluția codului, identifică erorile mai rapid și implementează practici de codare mai bune.
  • Protejează munca: Previne pierderea datelor prin stocarea sigură a istoricului proiectelor și a backup-urilor locale.
  • Simplifică fluxurile de lucru: Automatizează sarcinile repetitive, implementează fluxuri de lucru personalizate și îmbunătățește eficiența generală.

Utilizări comune ale Git:

  • Dezvoltare software
  • Managementul proiectelor
  • Urmărirea documentelor
  • Stocare de configurații

Exemple de utilizare a terminalului Git

Iată câteva exemple de comenzi Git comune pe care le puteți utiliza în terminal:

Inițializare

  • git init -> Inițializează un nou depozit Git în directorul curent.
  • git clone <url> -> Cloneză un depozit Git existent de la o adresă URL specificată.

Adăugarea și confirmarea modificărilor:

  • git add <nume_fișier> -> Adaugă fișierele specificate la stadiul de indexare pentru următoarea confirmare.
  • git add -> Adaugă toate fișierele modificate din directorul curent la stadiul de indexare.
  • git commit -m "Mesajul dumneavoastră" -> Comite modificările indexate cu un mesaj descriptiv.

Vizualizarea stării depozitului:

  • git status: -> Afișează starea fișierelor din directorul de lucru, indicând care fișiere sunt modificate, indexate sau neurmărite
  • git log: -> Afișează istoricul comiterilor din depozitul Git.

Gestionarea ramurilor:

  • git branch: -> Afișează o listă cu ramurile din depozitul Git.
  • git checkout <nume_ramură>: -> Schimbă la o ramură specificată.
  • git branch <nume_ramură>: -> Creează o nouă ramură numită <nume_ramură>.
  • git merge <nume_ramură>: Integrează modificările dintr-o altă ramură în ramura curentă.

Obținerea modificărilor din alte depozite:

  • git fetch: -> Descarcă cele mai recente modificări din depozitul Git la distanță.
  • git pull: -> Descarcă cele mai recente modificări din depozitul Git la distanță și le integrează în ramura curentă.

Depanare:

  • git reset HEAD~1: -> Anulează ultima confirmare.
  • git reflog: -> Afișează istoricul comenzilor Git executate.
